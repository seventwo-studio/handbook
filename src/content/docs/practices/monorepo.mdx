Every SevenTwo Studio project lives in a monorepo with the same top-level shape. Consistent structure means tooling, CI, and onboarding work the same way regardless of which product you are working on.

## Top-Level Layout

```
/
‚îú‚îÄ‚îÄ apps/
‚îÇ   ‚îú‚îÄ‚îÄ api/            # Hono API
‚îÇ   ‚îú‚îÄ‚îÄ mobile/         # Expo App (iOS + Android + Web)
‚îÇ   ‚îú‚îÄ‚îÄ ios/            # Native iOS (SwiftUI)
‚îÇ   ‚îú‚îÄ‚îÄ android/        # Native Android (Kotlin + Compose)
‚îÇ   ‚îú‚îÄ‚îÄ web/            # Astro Website
‚îÇ   ‚îî‚îÄ‚îÄ dashboard/      # TanStack Dashboard
‚îú‚îÄ‚îÄ packages/
‚îÇ   ‚îú‚îÄ‚îÄ ui/             # Shared React components (shadcn/ui base)
‚îÇ   ‚îú‚îÄ‚îÄ config/         # Shared tooling config (tsconfig, eslint, prettier)
‚îÇ   ‚îî‚îÄ‚îÄ types/          # Shared TypeScript types and Zod schemas
‚îú‚îÄ‚îÄ turbo.json
‚îú‚îÄ‚îÄ package.json        # pnpm workspace root
‚îî‚îÄ‚îÄ pnpm-workspace.yaml
```

Not every project uses every app slot. A project with only a mobile app and an API will have only `apps/api` and `apps/mobile`. Add slots as needed; never restructure the top-level shape.

## Tooling

| Tool | Purpose |
|------|---------|
| **pnpm workspaces** | Package manager and workspace linking |
| **Turborepo** | Task orchestration, caching, and pipeline ordering |
| **TypeScript project references** | Cross-package type checking |

## Shared Packages

### `packages/ui`

Base component library built on shadcn/ui. Consumed by `apps/web`, `apps/dashboard`, and any Expo web target.

```
packages/ui/
‚îú‚îÄ‚îÄ src/
‚îÇ   ‚îî‚îÄ‚îÄ components/     # shadcn/ui base components
‚îú‚îÄ‚îÄ package.json
‚îî‚îÄ‚îÄ tsconfig.json
```

### `packages/config`

Shared tooling presets. Never copy config files between apps‚Äîimport from here.

```

‚îú‚îÄ‚îÄ tsconfig/           # Base tsconfig presets (base, node, react)
‚îú‚îÄ‚îÄ eslint/             # Shared ESLint flat configs
‚îî‚îÄ‚îÄ prettier/           # Prettier config
```

### `packages/types`

API contracts, Zod schemas, and domain types shared across the full stack. The source of truth for request/response shapes used by both the API and its consumers.

```
packages/types/
‚îú‚îÄ‚îÄ src/
‚îÇ   ‚îú‚îÄ‚îÄ api/            # Zod schemas for API requests and responses
‚îÇ   ‚îî‚îÄ‚îÄ domain/         # Shared domain types (User, Product, etc.)
‚îú‚îÄ‚îÄ package.json
‚îî‚îÄ‚îÄ tsconfig.json
```
